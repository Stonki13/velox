IMPLEMENTATION NOTES
1. Build integration: new CMake option VELOX_BUILD_SANDBOX, default OFF, so CI and library consumers are unaffected. When ON, build examples/sandbox/ as target velox_sandbox.
2. Windowing/rendering: keep it dependency-light. Preferred: GLFW fetched via CMake FetchContent (pinned tag) + OpenGL 3.3 core for line rendering; if you judge a plain Win32+WGL window simpler and it keeps the code portable behind the window.h abstraction, that is acceptable, but the spec wants Windows AND Linux buildability, which favors GLFW. Render DebugLine segments with a single VBO rebuilt per frame (one draw call); this is fine up to ~100k lines.
3. Files per the spec: examples/sandbox/main.cpp, window.h/cpp, renderer.h/cpp, input.h/cpp, scenes.json. Text overlay: a tiny embedded bitmap font (e.g. 8x8 font baked into a header) — do NOT pull in a font library.

Controls per the spec: WASD camera move, Q/E rotate, Space adds a sphere at the cursor (raycast from camera through cursor to find the spawn point; spawn slightly above the hit), R reset, P pause, +/- substeps, F1-F4 presets.
